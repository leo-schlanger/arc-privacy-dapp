# Contributing to ArcShield

Thank you for your interest in contributing to ArcShield! We are building the future of privacy-first payments on the Arc Network.

## Code of Conduct
Please communicate with respect and professionalism. We are an inclusive community.

## How to Contribute

1.  **Fork the Repository**: Create your own fork of the code.
2.  **Create a Branch**: `git checkout -b feature/amazing-feature`
3.  **Commit Changes**: Use semantic commit messages (e.g., `feat: add new proof generation logic`).
4.  **Push to Branch**: `git push origin feature/amazing-feature`
5.  **Open a Pull Request**: Describe your changes clearly and link to any relevant issues.

## Development Standards

### Smart Contracts (Foundry)
- Run tests: `forge test`
- Format code: `forge fmt`
- Ensure all new functions have NatSpec comments.

### Frontend (Next.js)
- Run dev server: `npm run dev`
- Linting: `npm run lint`
- Use Tailwind CSS for styling.
- Ensure generic components are placed in `components/` and page-specific logic in `app/`.

## Reporting Bugs
Please use the GitHub Issues tab to report bugs. Include:
- Steps to reproduce
- Expected behavior
- actual behavior
- Screenshots (if applicable)
